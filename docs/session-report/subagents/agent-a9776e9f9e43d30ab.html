The bug

toolIndex[b.Tool.ID] = b.Tool registration only happens in the main-chain block. So when a tool_use occurs INSIDE a sidechain (a subagent running Bash/Read/etc.), its id is never registered, and the following tool_result inside that sidechain finds nothing in toolIndex and silently drops the result. Tool calls made by subagents are the common case, so this breaks real transcripts.

The fix

Register tool_use ids into toolIndex inside buildTurn itself, so BOTH main-chain and sidechain tool calls are registered. Then the main-chain loop only needs to seed Task subagent slots.

			turn.Blocks = append(turn.Blocks, model.Block{Type: model.BlockToolUse, Tool: buildToolCall(b)})

			toolIndex[tc.ID] = tc
