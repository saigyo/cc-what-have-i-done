1	# ccwhid — cc-what-have-i-done
2	
3	Turn a Claude Code session transcript into a browsable, self-contained static
4	HTML report. See what you prompted and what Claude Code did — and commit the
5	report into your repo as a documentation artifact.
6	
7	> **Status:** under active development. See
8	> [`docs/superpowers/plans/`](docs/superpowers/plans/) for the implementation plan.
9	
10	## Preview
11	
12	A full example — ccwhid rendering the very session that built it — is committed
13	at [`docs/session-report/index.html`](docs/session-report/index.html); clone the
14	repo and open it in a browser.
15	
16	| Report overview | Tool calls & diffs |
17	|:---:|:---:|
18	| [![ccwhid report overview](docs/images/report-top.png)](docs/session-report/index.html) | [![tool calls and diffs](docs/images/report-tool.png)](docs/session-report/index.html) |
19	
20	The report is one self-contained page: a jump-to-prompt sidebar, a live turn
21	filter, expand/collapse-all, and a light/dark toggle — all client-side, no
22	server.
23	
24	## Install
25	
26	```bash
27	go install github.com/saigyo/cc-what-have-i-done/cmd/ccwhid@latest
28	```
29	
30	Or build locally:
31	
32	```bash
33	go build -o ccwhid ./cmd/ccwhid
34	```
35	
36	## Usage
37	
38	```bash
39	ccwhid                      # browse sessions in an interactive TUI
40	ccwhid --latest             # render the most recent session
41	ccwhid --session <id>       # render a specific session (id or prefix)
42	ccwhid --session <id> --open
43	```
44	
45	The report is written to `./ccwhid-report/<session-short>/` by default (override
46	with `--out`). Open `index.html` in any browser — no server needed.
47	
48	### Browsing sessions
49	
50	Run `ccwhid` with no selector to open the interactive browser:
51	
52	- **Project list** — every project with sessions, most-recent first. `↑`/`↓`
53	  to move, `enter` (or `→`) to open a project, `q` to quit.
54	- **Session list** — the chosen project's sessions. `enter` selects, `←` (or
55	  `esc`) returns to the project list.
56	- Both lists **scroll** to fit any terminal size.
57	- By default only your **interactive sessions** are shown. Claude Code writes
58	  each Task subagent / code-review agent as its own transcript; press **`a`**
59	  in the session list to show (and hide) those agent sessions — they're marked
60	  with a `⟲`.
61	- Selecting a session opens an **options screen** to toggle subagents,
62	  redaction, and open-in-browser, and to type an output directory (blank uses
63	  the default) before generating.
64	
65	`--project <name>` opens the browser directly on that project's session list.
66	
67	![ccwhid project list](docs/images/tui-projects.png)
68	
69	*Pick a project, then a session. Agent transcripts are hidden until you press
70	`a`, then marked with `⟲`:*
71	
72	![ccwhid session list](docs/images/tui-sessions.png)
73	
74	### Flags
75	
76	| Flag | Description |
77	|------|-------------|
78	| `--session <id>` | Session id or unambiguous prefix to render |
79	| `--project <path\|name>` | Scope `--latest` to a project, or open the TUI on it (matches full path, basename, or unambiguous substring) |
80	| `--latest` | Render the most recent interactive session (skips agent transcripts) |
81	| `--out <dir>` | Output directory |
82	| `--title <str>` | Override the report title |
83	| `--include-subagents` | Include subagent work: inline Task sidechains and linked agent-session transcript pages under `subagents/` (default true) |
84	| `--no-redact` | Disable secret redaction |
85	| `--force` | Overwrite a non-empty output directory |
86	| `--open` | Open the report in a browser when done |
87	| `--usage` | Include a token-usage & estimated-cost section (default off) |
88	
89	## Redaction
90	
91	By default, ccwhid scrubs common secret shapes (AWS keys, API tokens, JWTs,
92	`KEY=[REDACTED:assignment] assignments) and rewrites your home directory to `~`.
93	This is best-effort defense-in-depth — **review generated reports before
94	committing them.** Disable with `--no-redact`.
95	
96	## What's included
97	
98	Your prompts, Claude's replies (rendered markdown + syntax-highlighted code),
99	tool calls with collapsible detail, `Edit`/`Write` diffs, collapsed thinking
100	blocks, and — optionally — nested subagent activity. Background-agent results
101	(e.g. a `Task` finishing) appear as their own "Agent" cards; linked subagent
102	sessions — found under the transcript's `<sessionId>/subagents/` directory —
103	render as separate pages under `subagents/`, linked from both the Agent tool
104	card and the result card. With `--usage`, their cost is rolled into the
105	totals and shown as an "of which subagents" line. System reminders and
106	attachments are omitted for readability.
107	
108	**Known limitation:** SDK-spawned sessions without parent linkage (e.g. some
109	review-hook runs) aren't attributed to a parent turn and so aren't included.
110	
111	## Token usage & cost
112	
113	Pass `--usage` (or tick "Include usage & cost" in the TUI) to add a collapsible
114	**Usage** card and per-turn cost badges. Token counts come straight from the
115	transcript's `usage` data and are exact; cost is an **estimate** from a built-in
116	Anthropic list-price table (dated in the report footnote) — unknown models show
117	tokens with cost `n/a`, and server-tool fees are not included. Prices are
118	embedded, so this works fully offline.
119	
120	These figures cover the rendered transcript plus any linked subagent sessions
121	(shown separately as "of which subagents"). SDK-spawned sessions without
122	parent linkage aren't attributed or included, which is why the total can
123	still differ from Claude Code's `/usage` session rollup.
